# Contributing

Thank you for helping improve this C programming collection.

## Before submitting a change

1. Keep each example focused on one clear concept.
2. Use standard C11 and portable standard-library headers.
3. Prefer descriptive variable names and concise comments that explain intent.
4. Check return values from input operations when adding new programs.
5. Do not commit compiled executables or editor-generated files.

## Validate your work

Run the compiler checks and build all examples:

```bash
make check
make
```

Also run any program you changed with representative input, including an edge case where appropriate.

## Pull requests

In the pull request description, briefly explain:

- what the program or change demonstrates;
- how you tested it; and
- any assumptions or known limitations.
